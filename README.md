# PersonaDB

> A personal data platform for collecting, organizing, and exploring my digital life.

## Overview

PersonaDB is a long-term personal project that brings together data from the services and applications I use into a single platform.

The project imports data from multiple sources, normalizes it into a consistent model, stores it in PostgreSQL, and provides applications and services for browsing, searching, analyzing, and visualizing that information.

While PersonaDB is built for my own use, it is also intended to be a portfolio project that demonstrates modern software architecture, backend development, frontend development, data engineering, and system design.

## Goals

- Import data from multiple external sources.
- Store data in a unified PostgreSQL database.
- Build reusable import pipelines.
- Provide web applications and APIs.
- Generate statistics, reports, and visualizations.
- Experiment with microservices and distributed architecture.
- Serve as a long-term platform that grows alongside my personal data.

## Repository Structure

```text
apps/       User-facing applications
services/   Background services and microservices
library/    Shared libraries and reusable code
data/       Local data, imports, exports, and development samples
docs/       Project documentation
scripts/    Development and maintenance utilities
```
